# 威海分行授信上报仪表盘 — 项目说明

## 项目是什么？

这是一个 **"威海分行法人客户授信上报仪表盘"** —— 简单说就是一个**数据看板网页**，用来展示威海各支行（荣成、乳山、文登、环翠、经开、营业部）授信上报的完成进度。

---

## App.jsx 核心功能拆解

这是整个项目的"大脑"，做了这几件事：

### 1. 数据来源

- 内置了一套**默认演示数据**（硬编码在代码里）
- 用户可以通过页面上的按钮**上传 CSV 文件**来替换数据
- 上传后的数据会存到浏览器的 **localStorage**（关掉浏览器再打开还在）

### 2. 数据处理（`dashboardData` 这个大计算块）

- 统计每个支行的**计划户数**和**实际上报户数**
- 计算**完成率**（实际 / 计划 × 100%）
- 按周汇总上报趋势
- 根据时间节点计算**计划进度**（比如3月底应完成30%、5月底应完成70%等）
- 判断当前是**超前完成**还是**落后进度**

### 3. 页面展示（4 大区域）

- **顶部 KPI 卡片**：总计划户数、已上报户数、整体完成率（绿色=达标，红色=落后）
- **柱状图**：各支行的计划 vs 实际对比，叠加完成率折线
- **饼图**：各支行已上报数量的占比分布
- **面积图**：按周展示上报数量走势
- **折线图**：计划完成率 vs 实际完成率的对比曲线

### 4. 导出功能

- 可以把整个仪表盘导出为 **PNG 图片**或 **PDF 文件**（方便发到群里或打印汇报）

## 两个 CSV 文件的格式

**plan.csv**（计划）：

```
机构名称,应上报授信户数
荣成,48
乳山,11
...
```

**date.csv**（实际上报记录）：

```
机构名称,客户名称,...,上报日期
荣成,荣成市人民医院,...,2026/2/14
...
```

## 怎么跑起来？

- `npm run dev` → 启动开发服务器，在浏览器打开 `http://127.0.0.1:5173` 就能看到仪表盘
- `npm run build` → 打包成静态网页文件（输出到 `dist/` 目录）
- 推送到 GitHub 后会自动通过 GitHub Actions 部署到 GitHub Pages

---

**一句话总结**：这是一个用 React + Recharts 搭建的银行授信上报进度看板，支持上传 CSV 数据、自动计算各支行完成情况、并能导出为图片/PDF。
